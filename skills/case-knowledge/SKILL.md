# Case Knowledge — Dynamic Document Analysis

## What This Skill Does

This skill builds case knowledge by reading documents in the workspace. The documents
are the source of truth — every fact, date, claim, and evidence link is extracted from
actual files, not hardcoded. This means:

- Works for any immigration IAC case, not just one specific client
- Knowledge stays current as new documents arrive
- Gaps are identified by what's absent from documents, not from a checklist
- Claims are only as strong as the documents that support them

## Architecture

```
case-knowledge/
├── SKILL.md                     # This file
├── case-strategy.md             # Strategic narrative (human-curated)
├── scripts/
│   ├── bootstrap.py             # Scan all docs → build knowledge graph
│   └── case_analyzer.py         # Gap analysis, coherence, strength
└── [generated by bootstrap.py]
    ├── case-facts.json          # Timeline extracted from documents
    ├── claims-matrix.json       # Claims with evidence mappings
    └── evidence-links.json      # Causal chains: exhibit → claim → prong
```

The JSON files are OUTPUT, not input. They're generated by scanning documents and
refined through agentic analysis. They can be regenerated at any time.

## Core Workflow

### Phase 1: Bootstrap

Scan all workspace documents to build the initial knowledge graph:

```bash
python scripts/bootstrap.py --workspace /path/to/case/files
```

This scans PDFs and DOCXs, extracts text, dates, names, receipt numbers, and docket
numbers, classifies documents by type, builds an initial timeline, and generates the
three JSON files plus a `bootstrap-report.md` flagging what was found, what's uncertain,
and what's missing.

### Phase 2: Agentic Deep Analysis

After bootstrap, Claude does the real analysis:

1. **Read the bootstrap report** to see what was auto-extracted
2. **Deep-read key documents** flagged as important (complaints, denials, attorney answers)
3. **Refine the knowledge graph** — correct extraction errors, add legal analysis, strengthen evidence links
4. **Run gap analysis** to identify what's missing and categorize HOW to fill each gap

### Phase 3: Ongoing Maintenance

As new documents arrive, re-run bootstrap with `--incremental` and re-run gap analysis.

## The Knowledge Files

### case-facts.json — What Happened

Every entry traces to a source document with a confidence level:

```json
{
  "events": [{
    "date": "2022-07-22",
    "description": "I-94 expired; unlawful presence begins accruing",
    "source_documents": ["path/to/evidence.pdf"],
    "confidence": "high",
    "significance": "critical"
  }],
  "unknowns": [{
    "question": "Exact date Julia first contacted Hayman Woodward",
    "why_it_matters": "Establishes when duty of care began",
    "category": "findable_in_documents"
  }]
}
```

### claims-matrix.json — What We're Arguing

Each claim maps to Lozada prongs with evidence strength FROM documents:

```json
{
  "claims": [{
    "id": "failed_advise_245k",
    "evidence": {
      "deficiency": {
        "supporting_docs": ["email_chain.pdf"],
        "strength": "strong",
        "reasoning": "Complete email chain shows zero mention of §245(k)"
      }
    },
    "gaps": [{
      "what": "No employer affidavit for consular processing",
      "category": "needs_external_input",
      "importance": "high"
    }]
  }]
}
```

### evidence-links.json — The Causal Chains

Links specific documents to specific claims through specific Lozada prongs, with page
references where possible.

## Gap Categories

Every unknown is classified into one of four categories:

| Category | What It Means | Action |
|----------|---------------|--------|
| `findable_in_documents` | Likely exists in workspace files, not yet extracted | Read the specific document more carefully |
| `discoverable_via_research` | Obtainable via CourtListener, web search, USCIS research | Use courtlistener or uscis-immigration-research skills |
| `needs_external_input` | Only the client or their attorney can provide | Flag for client to supply |
| `missing_and_critical` | Should exist but doesn't appear in workspace | Request from client, former counsel, or via FOIA |

## Running the Analyzer

```bash
python scripts/case_analyzer.py gap-analysis        # Missing evidence and unknowns
python scripts/case_analyzer.py coherence-check      # Cross-proceeding consistency
python scripts/case_analyzer.py strength-report      # Claim viability assessment
python scripts/case_analyzer.py timeline-verify      # Chronological consistency
python scripts/case_analyzer.py exhibit-coverage      # Exhibit utilization patterns
```

## Integration with Other Skills

Every skill should read case-knowledge before starting:

- **bar-complaint-reply-drafter**: Read claims-matrix for applicable claims and strength
- **motion-to-reopen**: Read case-facts for chronological narrative
- **evidence-manager**: Cross-reference evidence-links for exhibit-claim mappings
- **courtlistener**: Check gaps with `discoverable_via_research` for search targets
- **uscis-immigration-research**: Check gaps for statutory questions to research

## Key Principle: Flag What You Don't Know

The most valuable output is not what it finds — it's what's MISSING. Every analysis
should end with:

- What we know (with confidence levels and source documents)
- What we don't know (categorized by how to fix it)
- What we can find ourselves (research tasks for courtlistener/uscis skills)
- What we need from the client (specific input requests)
- What's missing and may not be obtainable (risk assessment)
